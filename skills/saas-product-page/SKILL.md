# SaaS Product Page Skill

Use this skill to create SaaS and AI-tool marketing pages that need product clarity, lead generation, signup, demo requests, or trial conversion.

## Inputs to Read First

- ICP: role, company size, problem maturity.
- Main job-to-be-done and current alternative.
- Product proof: screenshots, demo data, integrations, security posture.
- Conversion goal: trial, demo, waitlist, docs, pricing, contact sales.
- Technical constraints: APIs, data sources, compliance, deployment.

## Good Patterns

- The hero shows the workflow or output, not an abstract illustration.
- Copy names a measurable result and the system’s mechanism.
- Feature sections follow a workflow sequence.
- Security, integrations, and pricing are concrete enough for buyer confidence.
- Technical snippets are accurate, small, and purposeful.

## Bad Patterns

- “AI-powered platform for teams” with no job or user.
- UI mockups filled with lorem ipsum and nonsense metrics.
- Feature cards that all start with “Seamlessly”.
- Pricing hidden without any explanation of sales process.
- Charts that cannot be understood without color.

## Section Recipes

### 1. Product-Led Hero

Required:

- ICP-specific headline.
- One-sentence mechanism.
- Primary CTA and secondary CTA.
- Product visual: screenshot, diagram, terminal, or output sample.
- Trust row: customer type, security note, integration count, or quantified outcome.

### 2. Workflow Feature Section

Use three stages:

1. Connect/import/input.
2. Automate/analyze/generate.
3. Review/export/act.

Each stage needs a visual artifact and exact user benefit.

### 3. Use Cases

- 3 cards max for homepage.
- Each card names role, trigger situation, result, and product feature used.
- Avoid industry lists without tailored content.

### 4. Integrations

- Group by user mental model: data sources, destinations, collaboration, identity.
- Mark “coming soon” separately; do not present as live.
- Include manual upload/API fallback if available.

### 5. Security / Compliance

Cover:

- Data storage and retention summary.
- Permission model.
- Audit/export story.
- Compliance status or “not yet certified” honesty.

### 6. Pricing / Conversion

- Show tiers if self-serve.
- For sales-led, explain evaluation process and expected response.
- Include risk reducers: free trial, sandbox, migration support, cancellation policy.

## Component Recipes

### Product Screenshot Frame

- Browser/app chrome only if it clarifies context.
- Use realistic labels and data tied to the ICP.
- Highlight one action path per visual.

### Feature Card

- Title is outcome-led.
- Include “Before / After” or “Manual / Automated” contrast when possible.
- Keep body copy under 38 words.

### Comparison Table

- Compare against status quo, spreadsheets, agencies, internal scripts, or competitors only with defensible claims.
- Use row labels that match buyer criteria: setup time, data control, review workflow, export.

## QA Notes

- Validate that all screenshots/UI samples are legible at mobile width.
- Ensure CTAs map to actual routes or are marked as intended targets.
- Check security claims with owner/source before publishing.
- Confirm keyboard focus order through nav, hero CTA, pricing/demo forms.
- Run contrast checks on gradient or dark product-preview areas.

## Done Criteria

- Visitor understands who it is for, what task improves, how the product works, and how to start.
- Product visuals contain no placeholder nonsense.
- Security/integration/pricing sections reduce real objections rather than adding buzzwords.
