# Design Quality Gates

Human-grade output requires design gates before implementation, not only after QA.

## DESIGN.md Minimum Schema

`.ai-web/DESIGN.md` must define:

- brand mood
- target audience
- route and first-view intent
- section-by-section purpose
- typography scale
- color tokens
- spacing tokens
- grid and container rules
- imagery and icon art direction
- motion policy
- responsive behavior
- forbidden patterns
- claim/proof safety rules

## Candidate Strategy Gate

Exactly three candidates are generated by default. They must be meaningfully different strategies:

- `editorial-premium`
- `conversion-focused`
- `trust-minimal`

Project-specific design systems may map these labels to domain-specific equivalents, but candidates cannot be only color or spacing variants.

Each candidate comparison must include:

- strategy name
- first-view composition
- typography direction
- visual proof pattern
- CTA flow
- mobile behavior
- strengths
- tradeoffs
- risks

## Anti-Slop Gate

Reject or repair outputs with:

- fake numbers, fake awards, fake logos, or fake testimonials
- generic gradient blobs or decorative backgrounds without product meaning
- repetitive card grids without hierarchy
- overused centered hero plus three-card layout
- stock-like imagery that does not identify the actual product/service
- weak typography scale
- inconsistent spacing rhythm
- mobile fold with no clear value/action
- CTA overload
- inaccessible contrast or tap targets

## Visual Scoring

Visual critique scores must cover at least:

- first impression
- typography
- layout rhythm
- content credibility
- mobile polish
- interaction clarity
- brand differentiation
- originality

Passing requires:

- configured average score threshold
- configured per-axis minimum
- no critical accessibility failure
- mobile first-fold screenshot passes

Average score cannot hide a failed axis. A page with excellent desktop polish but poor mobile composition fails.

## Screenshot Gate

Responsive evidence must include first-fold and full-page screenshots for:

- 375px
- 390px
- 768px
- 1440px

Screenshots become inputs to visual critique and polish tasks.
